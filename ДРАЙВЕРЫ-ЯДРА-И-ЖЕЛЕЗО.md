# Car Thing (Superbird, Amlogic S905D2) — драйверы ядра ↔ реальное железо

> **Что это.** Единая сводная таблица всех значимых драйверов ядра Linux 4.9.113,
> их состояние (включён / выключен / модуль / спит), назначение каждого, и
> соотнесение с железом, которое **реально есть на плате**. Цель — чтобы не гадать
> «есть драйвер или нет» и «есть ли под него чип».
>
> Собрано: 2026-06-18. **Перепроверено на живом QN19 2026-06-18** (`/proc/config.gz` =
> 1298 `=y` / 10 `=m` / 2293 off; I2C raw-reads; ALSA; `/dev`; `/proc/modules`; DRM; thermal;
> IIO). Не искать заново — это точка правды по драйверам.

### ⚠️ Разрешённые противоречия конфигов
- **UDC гаджета `ff400000.dwc2_a`** управляется вендорским `CONFIG_AMLOGIC_USB=y` +
  `CONFIG_AMLOGIC_USB_DWC_OTG_HCD=y`, а **mainline `CONFIG_USB_DWC2` ВЫКЛЮЧЕН**. В live-аудите
  слово «DWC2» = название IP-блока, не mainline-драйвер. Это норма для Amlogic 4.9 vendor.
- `out-tdm-unmute/config` показывает `GE2D is not set` и `ST_ACCEL is not set` — это
  **старый срез (24.05)**. На текущем устройстве (defconfig + сборка 17.06) GE2D=y и ST_ACCEL=y.
- На устройстве почти всё **built-in (`=y`)**. Из модулей реально **загружены ТОЛЬКО**
  out-of-tree MFi (`apple_mfi_auth` + `apple_mfi_auth_i2c`, проверено в `/proc/modules` на QN19);
  `mali.ko` и все 10 in-tree `=m` НЕ загружены.

## Легенда состояний

| Знак | Значение |
|---|---|
| ✅ | `=y` built-in, **железо есть, работает** |
| 💤 | `=y`, **железо есть, ядро готово**, но не задействовано из userspace (спящая фича) |
| 🟡 | `=y`, но **чипа на плате НЕТ / чип молчит** → мёртвый груз |
| 🔌 | `=m` модуль (или out-of-tree `.ko`) |
| ⛔ | `is not set` — драйвер **выключен** в сборке |

## 11. Выключено / отсутствует — НЕ включать без пруфа железа

| Кандидат | Состояние | Почему нет |
|---|---|---|
| **Wi-Fi** (`CFG80211`/`MAC80211`/`WLAN`) | ⛔ полностью | В ядре драйвера нет; радиочип общий с BT, но WiFi-половина не разведена. Сеть — только USB NCM. **Тема закрыта** |
| **Ethernet** (`STMMAC_ETH`) | ⛔ | DTB disabled, нет PHY/разъёма |
| **Mali GPU** (`mali.ko`) | 🔌🟡 | Модуль собран, но **НЕ загружен**; `/dev/mali` отсутствует и нет DT-узла `mali`/`gpu` (проверено live) → не зависеть от GPU |
| `MESON_SM` (secure monitor отд.) | ⛔ | Не требуется |
| `IIO_ST_LSM6DSX`, `IIO_ST_ACCEL_3AXIS` | 🟡 | Драйверы есть — **чипов нет/молчат** (см. §7) |
| HDMI / VIN / IR-blaster / USB3-host | ⛔ | Нет трассировки/коннекторов на плате |
| `SECURITY_SELINUX` + `SECURITY_SMACK` | (скомпилены) | Политика не навешена в Buildroot → неактивны (вендор-наследие) |
| Linux `poweroff`/`halt` | — | **Запрещено**: уводит плату в Amlogic burn mode |

## Итог одной фразой
**Ядро закрыто под реальное железо:** дисплей, тач, кнопки, энкодер, T9015 line-out,
PDM-микрофоны, BT (BCM по UART), USB-gadget (NCM + UAC + storage), TMD2772, SARADC,
thermal×5, watchdog, RTC, HW-RNG/crypto, efuse, GE2D — **всё `=y` и работает**.
**Спят, ядро готово, нужен userspace:** USB-аудиокарта→A2DP-колонка, VAD/PDM-голос,
OSD2-оверлеи, медиа-декод, JTAG, DDR-телеметрия.
**Мёртвый груз (драйвер есть, чипа нет):** LIS2DH12, LSM6DSX, внешние аудио-кодеки, DVB,
Wi-Fi (драйвера даже нет), Mali (без userspace).
**Особый случай:** **MAX20332** — чип есть и отвечает, но **драйвера нет специально**;
управляем raw I2C из userspace (детект источника питания уже оттестирован).
