# Car Thing — Полная карта чипов и нераскрытых возможностей

**Дата:** 2026-05-22  
**Метод:** I2C scan, dmesg, DTB анализ, platform devices, исходники kernel

## Чипы на плате

### I2C шина 0
| Адрес | Чип | Что делает | Статус |
|-------|-----|-----------|--------|
| `0x2e` (UU) | **tlsc6x** | Тачскрин контроллер | ✅ используется |

### I2C шина 2
| Адрес | Чип | Что делает | Статус |
|-------|-----|-----------|--------|
| `0x18` (UU) | **LIS2DH12** | 3-осевой акселерометр (ST) | ⚡ есть драйвер, не используется |
| `0x35` | **MAX20332** | USB-C CC logic / мультиплексор | ❌ нет драйвера, probe failed |
| `0x39` (UU) | **TMD2772** | Ambient light + proximity (AMS) | ⚡ есть драйвер, не используется |

### I2C шина 3
| Адрес | Чип | Что делает | Статус |
|-------|-----|-----------|--------|
| `0x10` | **Apple MFi auth chip** | Аутентификация Apple MFi | ✅ драйвер есть, используется частично |

## MAX20332 — самое интересное открытие

**Maxim MAX20332** — USB-C CC logic + audio/USB мультиплексор.

Этот чип управляет USB-C портом устройства:
- Определяет ориентацию кабеля (CC1/CC2)
- Переключает между USB data и audio режимами
- Управляет VBUS
- Поддерживает USB-C Audio Accessory Mode (аналоговый аудио через USB-C)

**Текущий статус:** `modalias failure` — в kernel нет драйвера для MAX20332.  
Чип физически работает (иначе USB-C не функционировал бы), но управляется либо  
через прямой I2C из userspace, либо через GPIO.

**Потенциал:**
- Полный контроль над USB-C портом из userspace через I2C
- USB-C Audio Mode — подключение наушников напрямую к USB-C
- Определение типа подключённого устройства (кабель/зарядник/аксессуар)
- Управление VBUS (включать/выключать питание на порту)

**Как получить доступ прямо сейчас:**
```bash
# Прямой I2C доступ без драйвера
i2cget -y 2 0x35 0x00  # читать регистр 0x00
i2cdump -y 2 0x35      # дамп всех регистров
```

## Amlogic S905D2 (G12A) — нераскрытые блоки SoC

### ffe40000.bifrost — Mali-T60x GPU
**Статус:** инициализирован (`bifrost` platform device), но `/dev/mali*` отсутствует.  
Драйвер скомпилирован (`arm,malit60x`), но не создаёт device node.

**Что умеет:**
- 2 shader cores, частоты 125/285/400/500/666/850 MHz с DVFS
- OpenGL ES 2.0/3.0/3.1
- OpenCL 1.1/1.2 (через libmali)
- Vulkan (через Panfrost в upstream kernel)

**Потенциал:**
- GPU-ускоренный UI рендеринг (OpenGL ES вместо CPU software render)
- ML inference на GPU (через OpenCL или NNAPI)
- Аппаратное ускорение обработки изображений
- Шейдерные эффекты на дисплее

**Путь к активации:** нужен `libmali` blob от Amlogic или upstream Panfrost драйвер (kernel 6.6).

---

### ff940000.ge2d — 2D Graphics Engine
**Статус:** platform device есть, `/dev/ge2d` — проверить.

**Что умеет:**
- Hardware blit (копирование прямоугольников)
- Scale (масштабирование с фильтрацией)
- Rotate (поворот на 90/180/270°)
- Alpha blend (прозрачность)
- Color space conversion (RGB↔YUV)
- Mirror (отражение)

**Потенциал:**
- Аппаратный compositor для UI без нагрузки на CPU
- Масштабирование видео/изображений
- Поворот дисплея аппаратно
- Быстрый thumbnail generation

**Путь к активации:** Amlogic GE2D API через `/dev/ge2d` + ioctl. Исходники в kernel-stock.

---

### vdec / vcodec_dec — Hardware Video Decoder
**Статус:** platform device активен (`/dev/media.decoder`, `/dev/amvideo`).

**Что умеет (аппаратно):**
- H.264 до 4K@30fps
- H.265/HEVC до 4K@60fps
- VP9
- MPEG-2/4
- AV1 (частично)

**Потенциал:**
- Декодирование видео без нагрузки на CPU
- Стриминг видео с iPhone через AirPlay (если реализовать протокол)
- Воспроизведение локального видео

---

### amvenc_avc + hevc_enc — Hardware Video Encoder
**Статус:** platform devices активны.

**Что умеет:**
- H.264 encoding
- H.265/HEVC encoding

**Потенциал:**
- Запись экрана в видео
- Стриминг экрана по сети
- Сжатие видео с микрофонов

---

### ff900000.drm-vpu — Video Processing Unit
**Статус:** активен, используется для DRM/KMS.

**Дополнительные возможности:**
- `deinterlace` platform device — аппаратный деинтерлейсинг
- `rdma` — Register DMA, позволяет атомарно обновлять регистры VPU без CPU
- `meson-amvideom` — video overlay plane (второй видеослой поверх основного)
- `ppmgr` — post-processing manager (цветокоррекция, шарпнинг)
- `ionvideo` — ION video buffer management

**Потенциал:**
- Видео overlay: показывать видео поверх UI без копирования буферов
- Аппаратная цветокоррекция дисплея
- Double buffering через RDMA (атомарные обновления экрана)

---

### cvbsout / vdac — CVBS Video Output
**Статус:** platform device активен.

S905D2 имеет аналоговый CVBS (composite video) выход через VDAC.  
Пины на плату не выведены, но контроллер активен.  
**Потенциал:** при пайке — аналоговый видеовыход на внешний монитор/TV.

---

### amlogic,amlogic-pcie-v2 — PCIe контроллер
**Статус:** в DTSI есть, в живом DTB **отсутствует** — не инстанциирован.

PCIe Gen2 x1 физически в SoC. Вопрос только в том, выведены ли пины на плату.  
**Потенциал при пайке:** NVMe SSD, WiFi 6 карта, любое PCIe устройство.

---

### amlogic,amhdmitx — HDMI TX
**Статус:** в живом DTB есть (`amlogic,drm-amhdmitx`).

HDMI 2.0 + CEC (`amlogic,aocec-g12a`). Пины не выведены на разъём.  
**Потенциал при пайке:** полноценный HDMI выход с CEC управлением.

---

### amlogic,meson_irblaster — IR Blaster
**Статус:** в живом DTB есть.

Инфракрасный передатчик. Нужно проверить выведен ли пин на плату.  
**Потенциал:** управление любой IR техникой (TV, усилители, кондиционеры).

---

### ff642000.audiobus:tdma — TDM Audio Interface
**Статус:** platform device активен, но не используется.

TDM (Time Division Multiplexing) аудио интерфейс — для подключения внешних  
аудио кодеков, I2S устройств. T9015 DAC подключён через TDM.  
**Потенциал:** подключение внешнего аудио кодека через I2S/TDM.

---

### ff642000.audiobus:resample — Hardware Audio Resampler
**Статус:** platform device активен.

Аппаратный ресемплер — конвертация sample rate без нагрузки на CPU.

---

### UART C, D, E (неиспользуемые)
В SoC 5 UART контроллеров. Используются:
- ttyS0 (ff803000) — console
- ttyS1 (ffd24000) — Bluetooth

Остальные доступны через pinmux на GPIO пины.  
**Потенциал:** подключение внешних модулей (GPS, GSM, дополнительный MCU).

---

### SPI контроллеры (2 шт)
`amlogic,meson-g12a-spicc` — два SPI контроллера, не инстанциированы.  
**Потенциал:** SPI Flash, SPI дисплей, SPI сенсоры.

---

### I2C slave mode
`amlogic,meson-i2c-slave` — SoC может работать как I2C slave.  
**Потенциал:** другое устройство может обращаться к Car Thing как к I2C периферии.
